Sensor4Cities Overview

 

Sensor4Cities is a tool to disseminate scalar/multimedia data for single or groups of users, in the context of smart cities applications. Sensor4Cities allows users to access and share data in a simple, user-friendly and efficient manner through the use of social networks, web sites, including a mobile version of the web sites.

Sensor4Cities Arquitecture

 

The Sensor4Cities arquitecture consists of three components:

  • Front-End: provides access to the system. The user can access the system through Sensor4Cities website, mobile version and social newtworks.

  • Back-End: consists of two sub-components, database and web application, which are running in a server.

  • WMSNs: have three sub-components, job daemon, Base Station (BS) and nodes.

How to Request Data

 

The user can request and receive monitored information by using social networks, e.g., Twitter and Facebook, due to ther popularity, user-friendly interfaces and easy dissemination.

The user can request scalar data by using Twitter. In specific terms, the users have to pass through four steps, which are:

    What happens:

    (1) send a tweet to Twitter of Sensor4Cities, i.e., @sensor4cities.
    The tweet must contain the hashtag #sensor for requesting the scalar data or #multimedia for multimedia data. Thus, the web application can identify the request by using these tags.

    (2) BS broadcasts the request to WMSN nodes.

    (3) the nodes collect the multimedia/scalar data and send a reply.

    (4) a tweet is created by the tool as a reply to the user request.

    What the user sees:

    (1) send a tweet to Twitter of Sensor4Cities, i.e., @sensor4cities.
    The tweet must contain the hashtag #sensor for requesting the scalar data or #multimedia for multimedia data. Thus, the web application can identify the request by using these tags.

    (4) a tweet is created by the tool as a reply to the user request.
